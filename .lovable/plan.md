# PersonaSwap — лендинг в любом голосе

Сервис, в котором юзер вводит свой домен, выбирает личность (Хормози, Джобс, Черчилль, Карлин и т.д.) — и получает свой же лендинг, переписанный её голосом. С превью, скачиванием HTML и публичной ссылкой.

## Главный экран

- По центру — крупная поисковая строка для домена (`https://example.com`) и кнопка **Go**.
- Под ней — каталог личностей, сгруппированный по категориям-табам:
  - Бизнес-коучи (Hormozi, Tony Robbins, Brené Brown…)
  - Предприниматели (Jobs, Musk, Bezos, Branson, Sara Blakely…)
  - Актёры (Morgan Freeman, Keanu Reeves, Audrey Hepburn…)
  - Исторические личности (Churchill, Napoleon, Cleopatra, MLK…)
  - Стендаперы (Carlin, Chappelle, Hannah Gadsby…)
  - Писатели (Hemingway, Bukowski, Tolkien…)
  - Спортсмены, политики, музыканты, учёные…
- Каждая карточка: аватар-инициалы, имя, 1 строка о стиле. Поиск + фильтр по категории.
- Можно ввести **своё имя** в поле «Other persona» — стиль сгенерит AI на лету.

## Поток после нажатия Go

1. Левая панель «схлопывается» в сайдбар (анимация slide-left), справа открывается рабочая зона.
2. Сайдбар сохраняет: домен, выбранную личность, кнопку **Change persona**, кнопку **Go again**.
3. В рабочей зоне последовательно показываются статусы: *Fetching site → Extracting copy → Rewriting in voice of X → Rendering preview*.
4. Когда готово — превью в `iframe`, сверху панель: **Download HTML**, **Copy share link**, **Open in new tab**, переключатель *Original / Rewritten*.

## Что переписывается

Только текст + заголовки + meta `title`/`description`. Вёрстка, классы, стили, картинки и скрипты клиента остаются нетронутыми — скачанный HTML открывается так же, как оригинал, только с новым голосом.

## База личностей

Гибрид:
- **Курируемая база** (~120–150 имён) хранится как JSON в репо: `id`, `name`, `category`, `short_bio`, `voice_prompt` (детальное описание тона, лексики, ритма, типичных приёмов, запретов), `signature_phrases`, `avatar_color`.
- **Custom persona** — если юзер вводит имя, которого нет, edge-функция просит AI собрать профиль голоса по тому же шаблону, кэширует в БД и переиспользует.

## Сбор HTML с домена клиента

- Через коннектор **Firecrawl** (`scrape` с форматами `html` + `markdown`, `onlyMainContent: false`, `waitFor` для SPA).
- Edge-функция нормализует домен, запускает scrape, отдаёт обратно сырой HTML + извлечённые текстовые сегменты с указателями на их позицию (текстовые ноды + `<title>`, `<meta description>`, `alt`, `aria-label`).
- Лимит размера страницы; если ошибка/402 кредитов — понятное сообщение пользователю.

## Переписывание в голосе персоны

- Edge-функция `rewrite-landing` через Lovable AI Gateway (`google/gemini-2.5-flash` по умолчанию, `pro` для длинных страниц).
- Промпт: `voice_prompt` персоны + правила («сохраняй смысл и факты, не выдумывай цифры, держи длину ±20%, не трогай код/URL»).
- Текстовые сегменты пакуются батчами и переписываются за 1–2 вызова с JSON-выходом `{id → new_text}`.
- Затем сегменты вставляются обратно в исходный HTML по индексам — структура DOM сохраняется.

## Выдача результата

- **Превью**: srcdoc в iframe (sandbox), переключатель Original/Rewritten.
- **Download**: один `.html` файл, готовый к заливке.
- **Share link**: результат сохраняется в Lovable Cloud (таблица `rewrites`: domain, persona_id, html_original, html_rewritten, created_at, public_id), публичная страница `/r/:publicId` рендерит превью + кнопку скачать. RLS: создавать может кто угодно (анонимно), читать — только по публичному id.
- История последних 10 ребрендов в localStorage для быстрого возврата.

## Дизайн

Минималистичный, тёмный фон (Midnight Indigo), крупная типографика (Space Grotesk + DM Sans), много воздуха, мягкие переходы между состояниями (search → workspace).

## Технические детали

- Stack: React + Vite + Tailwind + shadcn (уже есть). Lovable Cloud для БД и edge-функций.
- Коннекторы: **Firecrawl** (scrape), **Lovable AI Gateway** (рерайтинг — встроено).
- Edge-функции:
  - `fetch-site` — Firecrawl scrape + извлечение текстовых сегментов.
  - `rewrite-landing` — батч-рерайт через AI Gateway, сборка финального HTML, сохранение в БД.
  - `generate-persona` — генерация voice profile для кастомных имён.
- Таблицы: `personas` (seed-данные + кастомные), `rewrites` (результаты с public_id).
- Rate-limit на edge-функциях по IP, валидация URL и имён через zod.
- Артефакты: HTML отдаётся напрямую blob-скачиванием, share — через публичный route.
